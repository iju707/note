Windows 다중 사용자(Multi-User) 환경이나 원격 데스크톱(RDP)을 사용하다 보면, 한쪽 세션에서 크롬이 실행 중일 때 다른 사용자의 화면에서 크롬이 뜨지 않거나 기존 창만 깜빡이는 문제가 발생합니다. 

<!-- truncate -->

이 문제는 크롬의 **사용자 데이터 디렉토리(User Data Directory)** 점유 방식 때문에 발생하는데요, 이를 해결하는 가장 확실한 방법들을 정리했습니다.

## 1. 문제의 원인
크롬은 보안과 데이터 무결성을 위해 **하나의 데이터 프로필을 한 번에 하나의 프로세스에서만 제어**하도록 설계되어 있습니다. 따라서 서로 다른 윈도우 세션에서 동일한 경로의 프로필을 호출하면 충돌이 발생하여 실행이 차단됩니다.

---

## 2. 해결 방법: 사용자별 프로필 경로 분리 (추천)
각 사용자 세션마다 독립적인 데이터 폴더를 지정해주면 충돌을 완벽히 피할 수 있습니다. 


1. 크롬 바로가기 아이콘 우클릭 > **속성** 선택.
2. **바로가기** 탭의 **대상(T)** 필드 끝에 아래 파라미터를 추가합니다. (기존 경로 뒤에 한 칸 띄우고 입력)

```bash
--user-data-dir="C:\ChromeProfile\%USERNAME%"
```
 * 설명: %USERNAME% 환경 변수를 사용하면 접속한 윈도우 계정명에 따라 자동으로 폴더가 생성됩니다.
3. 기타 조치 사항
위 방법으로도 해결되지 않는다면 아래 내용을 점검해 보세요.
백그라운드 앱 실행 종료
이전 세션의 크롬이 완전히 죽지 않고 프로세스에 남아있어 새 세션 실행을 방해하는 경우입니다.
 * Ctrl + Shift + Esc (작업 관리자) 실행.
 * chrome.exe 관련 프로세스를 모두 작업 끝내기.
 * 크롬 설정에서 'Google Chrome 종료 후에도 백그라운드 앱을 계속 실행' 옵션을 비활성화합니다.
4. 요약
가장 간단한 해결책은 **실행 옵션(--user-data-dir)** 을 변경하는 것입니다. 원격 접속 환경에서 크롬이 한쪽에서만 뜨는 현상으로 고생 중이시라면 바로 적용해 보세요!

:::tip
만약 Docker 환경에서 크롬 헤드리스(Headless) 등을 멀티 세션으로 돌릴 때도 위와 같이 데이터 디렉토리를 다르게 지정하는 방식이 유효합니다.
:::
