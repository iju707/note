이번 포스트에서는 Docusaurus를 설치하고 블로그 전용으로 설정한 뒤, GitHub Actions를 통해 자동 배포하는 방법을 알아봅니다.

<!-- truncate -->

## 1. Docusaurus 설치

`npx`를 사용하여 Docusaurus 프로젝트를 쉽게 생성할 수 있습니다.

```bash
npx create-docusaurus@latest my-website classic
```

설치가 완료되면 생성된 프로젝트 폴더로 이동하여 작업을 시작합니다.

```bash
cd my-website
```

## 2. 블로그 전용으로 설정 변경

Docusaurus의 기본 템플릿은 문서(Docs)와 블로그가 함께 활성화되어 있습니다. 문서 기능을 끄고 블로그 기능만 사용하려면 `docusaurus.config.ts` 파일을 다음과 같이 수정합니다.

`presets` 배열 내부의 `classic` 설정에서 `docs` 옵션을 `false`로 지정하고, `blog`의 `routeBasePath`를 `/`로 변경합니다.

```typescript title="docusaurus.config.ts"
export default {
  // ... 생략 ...
  presets: [
    [
      'classic',
      {
        docs: false, // 문서 기능 완전 비활성화
        blog: {
          routeBasePath: '/', // 블로그 게시물을 사이트의 메인(/)으로 이동
          showReadingTime: true,
        },
        theme: {
          customCss: './src/css/custom.css',
        },
      },
    ],
  ],
  // ... 생략 ...
};
```

또한 메인 화면이 `/` 경로의 블로그로 연결되므로, 상단 네비게이션 바(`themeConfig.navbar.items`)에 있던 `/blog` 경로로 가는 링크는 제거해 주는 것이 좋습니다.

## 3. GitHub Actions 적용 가이드

작업한 블로그를 GitHub Pages에 자동으로 배포하기 위해 GitHub Actions 워크플로를 설정합니다.
프로젝트 최상단에 `.github/workflows/deploy.yml` 파일을 만들고 아래 코드를 작성합니다.

```yaml title=".github/workflows/deploy.yml"
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - master # 또는 main 브랜치 이름에 맞게 설정

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    name: Build Docusaurus
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 24 # 사용 중인 Node 버전에 맞추어 작성
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build website
        run: npm run build

      - name: Upload Build Artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: build

  deploy:
    name: Deploy to GitHub Pages
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

마지막으로 GitHub 리포지토리의 **Settings > Pages > Build and deployment** 메뉴로 이동하여 Source를 **GitHub Actions**로 변경합니다. 

이렇게 설정해 두면 이후 `master` 브랜치에 코드가 푸시될 때마다 GitHub 워크플로가 가동되어 블로그 사이트가 새로 배포됩니다.
